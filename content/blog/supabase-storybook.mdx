Storybook transforms component development. Instead of navigating through an entire application to test a button variant, you can develop and test components in isolation with hot reloading.

I set up Storybook as a dedicated app within our turborepo to improve the component development workflow.

## The Setup

Adding Storybook to an existing monorepo requires careful configuration to work with:

- Our existing Tailwind setup and design tokens
- The theming system (light, dark, deep-dark)
- TypeScript and path aliases
- Component dependencies from `packages/ui`

```js title=".storybook/main.js"
module.exports = {
  stories: ['../packages/ui/**/*.stories.@(ts|tsx)'],
  addons: [
    '@storybook/addon-essentials',
    '@storybook/addon-themes',
  ],
  framework: {
    name: '@storybook/nextjs',
    options: {},
  },
}
```

## Key Features

The Storybook setup includes:

- **Theme switching** to preview components in all themes
- **Viewport testing** for responsive behavior
- **Controls panel** for interactive prop manipulation  
- **Auto-generated docs** from TypeScript types
- **Accessibility testing** with a11y addon

## Story Structure

Each component has stories that demonstrate:

```tsx title="Button.stories.tsx"
import type { Meta, StoryObj } from '@storybook/react'
import { Button } from './Button'

const meta: Meta<typeof Button> = {
  component: Button,
  tags: ['autodocs'],
}

export default meta
type Story = StoryObj<typeof Button>

export const Primary: Story = {
  args: {
    variant: 'default',
    children: 'Primary Button',
  },
}

export const AllVariants: Story = {
  render: () => (
    <div className="flex gap-4">
      <Button variant="default">Default</Button>
      <Button variant="outline">Outline</Button>
      <Button variant="ghost">Ghost</Button>
      <Button variant="destructive">Destructive</Button>
    </div>
  ),
}
```

## Impact

Storybook has improved our development workflow by:

- Enabling **faster iteration** on component designs
- Providing a **visual regression baseline** for testing
- Serving as **living documentation** that's always up to date
- Making it easier for **contributors** to understand our component library

The setup later evolved into our full design system site, which uses Storybook concepts but with a custom implementation using iframes for better isolation.
