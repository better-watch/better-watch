Navigation is the backbone of any dashboard. As Supabase grew from a handful of features to a comprehensive platform, the original sidebar became cramped and hard to navigate. I led the sidebar redesign effort.

## The Problem

The original sidebar had grown organically:

- Too many top-level items
- Inconsistent iconography
- No visual hierarchy between sections
- Mobile experience was an afterthought
- No support for collapsing/expanding

## Design Goals

The new sidebar needed to:

1. **Scale** as we add more features
2. **Collapse** for more screen real estate
3. **Group** related items logically
4. **Adapt** to mobile with a sheet/drawer pattern
5. **Persist** user preferences

## Implementation

### Collapsible Sidebar

The sidebar can collapse to icon-only mode, expanding on hover or click:

```tsx title="Sidebar state management"
const [isCollapsed, setIsCollapsed] = useState(() => {
  return localStorage.getItem('sidebar-collapsed') === 'true'
})

useEffect(() => {
  localStorage.setItem('sidebar-collapsed', String(isCollapsed))
}, [isCollapsed])
```

### Mobile Sheet Pattern

On mobile, the sidebar becomes a sheet that slides in from the left:

```tsx title="Responsive sidebar"
function Sidebar() {
  const isMobile = useMediaQuery('(max-width: 768px)')
  
  if (isMobile) {
    return (
      <Sheet>
        <SheetTrigger asChild>
          <Button variant="ghost" size="icon">
            <MenuIcon />
          </Button>
        </SheetTrigger>
        <SheetContent side="left">
          <SidebarContent />
        </SheetContent>
      </Sheet>
    )
  }
  
  return <SidebarContent />
}
```

### Active State Handling

Navigation items show different stroke widths when active, providing clear visual feedback:

```tsx title="Nav item with active state"
<NavItem 
  href="/project/[ref]/editor"
  icon={<TableIcon strokeWidth={isActive ? 2 : 1.5} />}
>
  Table Editor
</NavItem>
```

## Project & Org Switching

The sidebar header includes a popover for switching between projects and organizations without leaving the current page:

- Recent projects for quick access
- Organization grouping
- Search for large accounts
- Truncation for long project names

## Result

The new sidebar:

- Reduces cognitive load with better grouping
- Reclaims screen space when collapsed  
- Works seamlessly on mobile
- Scales to accommodate new features
- Respects user preferences
