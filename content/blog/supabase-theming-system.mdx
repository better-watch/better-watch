Theming might seem simple—just swap some colors, right? But building a robust theming system for a platform with multiple applications, component states, and accessibility requirements is genuinely complex.

## The Architecture

The theming system I built for Supabase uses CSS custom properties (variables) as the foundation, with Tailwind CSS classes mapping to these variables. This gives us:

- **Runtime theme switching** without JavaScript bundle changes
- **System preference detection** for automatic dark/light mode
- **Per-component theme overrides** when needed
- **Consistent tokens** across all applications

```css title="Theme token structure"
:root {
  --background-default: 0 0% 100%;
  --foreground-default: 222 47% 11%;
  --border-default: 214 32% 91%;
  /* ... hundreds more tokens */
}

[data-theme="dark"] {
  --background-default: 222 47% 11%;
  --foreground-default: 210 40% 98%;
  --border-default: 215 28% 17%;
}
```

## Design Tokens

The system uses a semantic token approach:

- **Color tokens** for backgrounds, foregrounds, borders, and accents
- **Spacing tokens** that scale consistently
- **Typography tokens** for font sizes, weights, and line heights
- **Animation tokens** for consistent motion

Each token has semantic meaning. Instead of `bg-gray-900`, we use `bg-background-default`. This abstraction is what allows themes to work—the meaning stays the same, but the actual color values change.

## Implementation Details

The theming system integrates with:

- **next-themes** for SSR-safe theme switching
- **Tailwind CSS** via a custom plugin for our tokens
- **Radix UI** for accessible component primitives
- **Our component library** for consistent usage patterns

```tsx title="Theme provider setup"
import { ThemeProvider } from '@supabase/ui'

export function App({ children }) {
  return (
    <ThemeProvider defaultTheme="system" enableSystem>
      {children}
    </ThemeProvider>
  )
}
```

## Impact

The theming system now powers:

- Studio (the main dashboard)
- The marketing website
- Documentation site
- Design system site
- All internal tools

Users can switch between light and dark modes seamlessly, with their preference persisted and respected across sessions.
