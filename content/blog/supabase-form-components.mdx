Forms are everywhere in the dashboard. I built form components and patterns for consistent, validated data entry.

## FormItemLayout

A layout component for form fields:

```tsx title="FormItemLayout usage"
<FormItemLayout
  label="Database Name"
  description="Choose a name for your database"
  error={errors.name?.message}
>
  <Input {...register('name')} />
</FormItemLayout>
```

### Features

- Consistent label placement
- Description text
- Error message display
- Optional indicators
- Before/after labels

## React Hook Form Integration

Migrated forms to react-hook-form:

### New Project Form

```tsx title="New project form"
const { register, handleSubmit, formState } = useForm<ProjectForm>()

return (
  <form onSubmit={handleSubmit(onSubmit)}>
    <FormField
      label="Project Name"
      {...register('name', { required: true })}
      error={formState.errors.name}
    />
  </form>
)
```

### Data API Settings

PostgREST settings using react-hook-form:

- Schema selection
- Max rows configuration
- Additional options
- Validation feedback

## Sheet Component

Form panels using Sheet:

```tsx title="Sheet for forms"
<Sheet>
  <SheetTrigger>Edit</SheetTrigger>
  <SheetContent>
    <SheetHeader>Edit Settings</SheetHeader>
    <form>...</form>
  </SheetContent>
</Sheet>
```

## Validation Patterns

Consistent validation:

```typescript title="Validation schema"
const schema = z.object({
  name: z.string().min(3).max(50),
  email: z.string().email(),
  region: z.enum(['us-east', 'eu-west']),
})
```
