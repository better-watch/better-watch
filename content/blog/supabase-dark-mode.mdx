Developers love dark mode. I implemented and maintained dark mode across the Supabase ecosystem.

## Light Mode Fixes

When light mode was added, many issues emerged:

- Roles showing black text on dark backgrounds
- Monaco editor themes not matching
- Checkbox colors incorrect
- Scrollbar styling issues

I fixed these systematically across the dashboard.

## Theme System

The theme system supports:

- System preference detection
- Manual toggle
- Persistence across sessions
- Real-time switching

```typescript title="Theme detection"
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)')

prefersDark.addEventListener('change', (e) => {
  setTheme(e.matches ? 'dark' : 'light')
})
```

## Monaco Dark Theme

Custom dark theme for the SQL editor:

- Syntax highlighting colors
- Background matching dashboard
- Cursor and selection colors
- Line number styling

## Component Styling

Each component needed dark mode support:

### Inputs

```css title="Dark mode inputs"
.input {
  background: var(--bg-secondary);
  border-color: var(--border-default);
  color: var(--text-primary);
}
```

### Modals

Overlay backgrounds with proper opacity and blur.

### Charts

Tooltip backgrounds fixed for dark mode visibility.

## Scrollbar Styling

Custom scrollbars for dark mode:

```css title="Dark scrollbars"
::-webkit-scrollbar-track {
  background: var(--bg-primary);
}
::-webkit-scrollbar-thumb {
  background: var(--bg-tertiary);
}
```

## Theme Switcher

The theme switcher in account settings:

- Visual preview of themes
- Instant application
- Persisted preference

## Deep Dark Theme

Beyond standard dark mode, I added the "Deep Dark" theme variant:

- Even darker backgrounds
- Higher contrast text
- OLED-friendly blacks

## Impact

Dark mode support:

1. Reduces eye strain
2. Saves battery on OLED
3. Matches developer preferences
4. Improves focus

Dark mode is table stakes for developer tools.
