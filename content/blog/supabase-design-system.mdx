Building a design system for a rapidly growing open-source platform is a unique challenge. When I joined Supabase, components were scattered across applications with inconsistent styling and behavior. I led the effort to consolidate everything into a cohesive, documented design system.

## The Challenge

Supabase has multiple applicationsâ€”the main marketing site, Studio (the dashboard), documentation, and various internal tools. Each had evolved independently, leading to:

- Duplicated components with subtle differences
- Inconsistent spacing, typography, and color usage
- No single source of truth for component APIs
- Difficulty onboarding new developers

## The Solution

I created a dedicated design system site that serves as both documentation and a living playground for all our components. The key features include:

- **Interactive component demos** using iframes for isolation
- **Comprehensive API documentation** auto-generated from TypeScript props
- **Theming documentation** showing how to customize the system
- **Pattern library** demonstrating common UI patterns

## Technical Highlights

The design system is built on Radix UI primitives, styled with Tailwind CSS, and uses our custom design tokens. This combination gives us:

```tsx title="Example: Button component"
import { Button } from '@supabase/ui'

export function Example() {
  return (
    <Button variant="primary" size="md">
      Get Started
    </Button>
  )
}
```

Every component supports our theming system out of the box, automatically adapting to light, dark, and deep-dark modes.

## Impact

The design system has become the foundation for all Supabase UI development. New features ship faster because developers can compose from existing, tested components rather than building from scratch.
