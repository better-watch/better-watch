As Supabase grew to support larger teams, the organization-level experience needed restructuring. I redesigned the org layout to better support team workflows.

## The Changes

### New Layout Structure

Organization pages now have a dedicated layout:
- Consistent sidebar across org pages
- Clear navigation between org settings
- Better hierarchy between org and project levels

### Navigation Improvements

- Org-level menu items grouped logically
- Quick switching between organizations
- Breadcrumbs showing org > project hierarchy

### Settings Organization

Org settings restructured into clear categories:
- **General**: Name, slug, avatar
- **Team**: Members, roles, invites
- **Billing**: Plans, usage, invoices
- **Security**: SSO, audit logs

## Technical Implementation

Using a shared layout wrapper:

```tsx title="Org layout"
export function OrgLayout({ children }) {
  const org = useCurrentOrg()
  
  return (
    <div className="flex">
      <OrgSidebar org={org} />
      <main className="flex-1">
        <OrgHeader org={org} />
        {children}
      </main>
    </div>
  )
}
```

### getLayout Pattern

Pages can opt into the org layout:

```tsx title="Page with org layout"
SettingsPage.getLayout = (page) => (
  <OrgLayout>{page}</OrgLayout>
)
```

## Result

Teams can now navigate organization settings more intuitively, with a clear separation between org-level and project-level concerns.
