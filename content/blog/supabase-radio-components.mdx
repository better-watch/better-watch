Radio buttons are common in forms, but the standard HTML radio doesn't cut it for modern UIs. I built two new radio group components for the Supabase design system.

## RadioGroupStacked

A vertical stack of radio options with better styling:

```tsx title="RadioGroupStacked usage"
<RadioGroupStacked value={selected} onValueChange={setSelected}>
  <RadioGroupStackedItem value="free" label="Free Tier" description="Perfect for hobby projects" />
  <RadioGroupStackedItem value="pro" label="Pro" description="For production applications" />
  <RadioGroupStackedItem value="team" label="Team" description="For growing teams" />
</RadioGroupStacked>
```

### Features

- Clear visual hierarchy
- Description text support
- Disabled states
- Keyboard navigation
- Focus indicators

## RadioGroupCard

Card-style radio selection for more visual choices:

```tsx title="RadioGroupCard usage"
<RadioGroupCard value={region} onValueChange={setRegion}>
  <RadioGroupCardItem value="us-east" icon={<FlagUS />} label="US East" />
  <RadioGroupCardItem value="eu-west" icon={<FlagEU />} label="EU West" />
  <RadioGroupCardItem value="ap-south" icon={<FlagIN />} label="AP South" />
</RadioGroupCard>
```

### Use Cases

- Region selection
- Plan selection
- Instance size selection
- Any choice with visual representation

## FormItem Integration

Both components integrate with the form system:

```tsx title="Form integration"
<FormItem>
  <FormLabel>Select Plan</FormLabel>
  <RadioGroupStacked {...field}>
    {plans.map(plan => (
      <RadioGroupStackedItem key={plan.id} {...plan} />
    ))}
  </RadioGroupStacked>
  <FormMessage />
</FormItem>
```

## Styling Improvements

Along with the new components:
- Fixed lint warnings across the codebase
- Improved focus/highlight states
- Better contrast ratios

## Result

These components are now used throughout the dashboard for plan selection, region picking, and other multi-choice scenarios.
