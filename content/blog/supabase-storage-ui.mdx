Supabase Storage provides S3-compatible object storage. I worked on the dashboard UI for managing files and folders.

## File Management

The storage UI needed to handle common operations:

- Uploading files (single and batch)
- Creating folders
- Renaming files and folders
- Moving items between folders
- Deleting with confirmation

## Folder Rename Validation

A subtle but important fixâ€”preventing invalid characters in folder names:

```typescript title="Folder rename validation"
function validateFolderName(name: string) {
  if (name.includes('/') || name.includes('\\')) {
    throw new Error('Folder names cannot contain slashes')
  }
  return name.trim()
}
```

This prevents broken paths and confusion.

## Error Handling

Storage operations can fail for various reasons:

- File too large
- Invalid file type
- Permission denied
- Network issues

I improved toast messages to be actionable:

```typescript title="Error toast messages"
toast.error({
  title: 'Upload failed',
  description: 'File exceeds 50MB limit. Try compressing or splitting.',
  action: {
    label: 'Learn more',
    onClick: () => openDocs('/storage/limits'),
  },
})
```

## Empty States

When buckets or folders are empty, users need guidance:

- Clear messaging about what goes here
- Quick action buttons to upload
- Links to documentation
- Visual indication of drag-and-drop support

## Performance

Storage UIs can get slow with many files:

- Pagination for large buckets
- Lazy loading of file previews
- Virtual scrolling for long lists
- Optimistic UI updates

## Integration with Policies

Storage policies control access. The UI surfaces policy status:

- Public/private bucket indicators
- Policy configuration links
- Access denied explanations
- Quick policy templates

## Impact

Storage UI improvements help users:

1. Manage files without CLI knowledge
2. Understand errors and fix them
3. Navigate large buckets efficiently
4. Configure access policies correctly

A good storage UI makes object storage accessible to everyone.
