Analytics shouldn't require invasive tracking. I led the implementation of privacy-first analytics across Supabase, eventually migrating to Plausible.

## The Problem with Traditional Analytics

Traditional analytics tools:

- Drop cookies without consent
- Track users across sites
- Create privacy compliance headaches
- Require cookie banners that hurt UX

## Cookie-Free Analytics

Early on, I removed tracking cookies and improved privacy:

```typescript title="Cookie-free page tracking"
async function trackPageView() {
  await fetch('/api/telemetry', {
    method: 'POST',
    body: JSON.stringify({
      event: 'page_view',
      path: window.location.pathname,
      // No user ID, no cookies
    }),
  })
}
```

This respects user privacy while still providing aggregate insights.

## Server-Side Events

For authenticated users, server-side tracking provides better data without client-side tracking:

```typescript title="Server-side event tracking"
// Sign up event - server side
Telemetry.identify({
  userId: user.id,
  traits: {
    plan: subscription.plan,
    // Only essential business data
  },
})
```

## Plausible Integration

In 2023, I integrated Plausible for privacy-first analytics:

- **No cookies** required
- **GDPR compliant** out of the box
- **Lightweight** (~1KB script)
- **Open source** and self-hostable

```typescript title="Plausible setup"
// Added to telemetry for page views and events
import { trackEvent } from '@plausible/tracker'

trackEvent('signup', {
  props: { plan: 'pro' }
})
```

## What We Track

With privacy-first analytics, we focus on:

- **Aggregate page views** (not individual sessions)
- **Feature usage** (what's being used)
- **Error rates** (what's breaking)
- **Conversion events** (signups, upgrades)

We explicitly don't track:

- Individual user journeys
- Personal information
- Cross-site behavior
- Third-party data

## HubSpot Integration

For marketing automation, I added HubSpot integration with privacy controls:

- Profile syncing only with consent
- Minimal data transfer
- Clear opt-out mechanisms

## Results

Privacy-first analytics provides:

1. **Cleaner UX**: No cookie banners needed
2. **User trust**: Respecting privacy builds goodwill
3. **Compliance**: GDPR/CCPA friendly by default
4. **Accurate data**: No ad blockers blocking tracking

You don't need to invade privacy to understand your users.
